# v5 Self-Debate Experiment

## Hypothesis

Does a structured adversarial debate protocol — where an ml-critic and ml-defender each independently evaluate a methodology case, then an orchestrator adjudicates — produce better ML methodology verdicts than a single-pass baseline? v5 tests this across five conditions (isolated_debate, multiround, forced_multiround, ensemble, baseline) on a 110-case benchmark (80 critique, 30 defense_wins; 42 hard, 68 medium) generated by the ARCH-1 pipeline. The primary metric is fair-comparison lift (fc_lift ≥ +0.10 on IDR/IDP/DRQ/FVC dimensions where baseline has equal structural agency). A secondary hypothesis tests whether adversarial structure adds beyond compute volume alone (debate vs. ensemble, H2).

## Pipeline

```
Phase 0   Setup — normalize_cases.py, benchmark_cases.json
Phase 1   CASE_VERIFIER — validate benchmark_cases.json (110 keep)
Phase 2   HYPOTHESIS.md + BENCHMARK_PROMPTS.md
Phase 3   Pre-registration — PREREGISTRATION.json + evaluation_rubric.json (locked)
Phase 4   Protocol self-review — CRITIQUE, DEFENSE, DEBATE, EXECUTION_PLAN
Phase 5   Scoring engine — self_debate_poc.py
Phase 5.5 Difficulty gate — 10-case stratified pilot + variance gate (BLOCKER)
Phase 6   Full benchmark run — all 5 conditions × 110 cases × n_runs
Phase 7   Scoring — v5_results.json, sensitivity analysis, variance analysis
Phase 8   Analysis — fc_lift by condition and stratum, H1/H2/H3 verdicts
Phase 9   Cross-vendor scoring — generalizability check
Phase 10  Report
Phase 11  Final artifacts + coherence audit
```

## Scope

**In scope:** Binary-verdict cases (critique_wins / defense_wins). Five structured conditions on a 110-case synthetic benchmark.

**Out of scope:** Mixed-verdict cases (zero in ARCH-1 — the mechanism claim cannot be generalized to calibrated judgment under genuine ambiguity from this benchmark). Real-world methodology cases. Model families other than claude-sonnet-4-6 (Phase 9 cross-vendor check addresses generalizability).

## Phase 5.5 Gate

Phase 5.5 runs a stratified 10-case pilot (5 medium, 5 hard) using claude-haiku-4-5 at 3 runs per case before Phase 6 begins. Two blockers: (1) ≥ 6/10 hard cases must score mean < 0.55; (2) within-case SD ≤ 0.10 (else n_runs_per_case increases from 3 to 5). Phase 6 does not start until both gates pass.
